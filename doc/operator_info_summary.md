# 运营商信息资料库字段总结

## 字段说明

### MCC (Mobile Country Code)
- **定义**：移动国家代码，由ITU-T分配给每个国家或地区
- **格式**：3位数字字符串
- **示例**：中国为"460"，美国为"310"
- **必填**：是

### MNC (Mobile Network Code)
- **定义**：移动网络代码，标识特定运营商
- **格式**：2-3位数字字符串，根据国家不同长度可能不同
- **示例**：中国移动在中国的MNC为"00","02","07"
- **必填**：是

### long_name
- **定义**：运营商的完整名称
- **格式**：字符串
- **示例**："China Mobile"，"Verizon"
- **必填**：是

### short_name
- **定义**：运营商的简称
- **格式**：字符串
- **示例**："CMCC"，"AT&T"
- **必填**：是

### 国家
- **定义**：运营商所在国家或地区的中文名称
- **格式**：字符串
- **示例**："中国"，"美国"
- **必填**：是

### 国家（英文）
- **定义**：国家或地区的ISO 3166-1字母代码
- **格式**：大写英文字母
- **示例**："CN"，"US"
- **必填**：是

### 国家编号
- **定义**：国家或地区的国际电话区号
- **格式**：数字字符串
- **示例**：中国为"+86"，美国为"+1"
- **必填**：是

### 号码长度
- **定义**：手机号码的总长度（不含国家代码）
- **格式**：整数
- **示例**：中国为11位，美国为10位
- **必填**：是

### 号码前缀
- **定义**：手机号码的前几位数字，用于区分不同的运营商或服务类型
- **格式**：字符串列表
- **示例**：中国为["139", "138", "137", ...]
- **必填**：是

## 数据录入规则

1. **MCC必须唯一**：每个国家/地区只能有一个MCC
2. **MNC不能重复**：同一国家内的不同运营商应有唯一的MNC
3. **国家编码标准化**：使用ISO 3166-1标准的国家代码
4. **号码前缀规范**：确保前缀与实际运营商使用的号码范围一致
5. **多值处理**：号码前缀可以包含多个值，用英文逗号分隔

## 示例

```json
{
  "mcc": "460",
  "mnc": "00",
  "long_name": "China Mobile",
  "short_name": "CMCC",
  "国家": "中国",
  "国家（英文）": "CN",
  "国家编号": "+86",
  "号码长度": 11,
  "号码前缀": ["139", "138", "137"]
}
```

## 使用方法

将上述字段的数据按照格式要求填写到资料库中，确保所有必填项都已正确填写。
