# 运营商信息配置说明

## 概述

本文档详细说明了如何配置和维护运营商信息，包括MCC、MNC、国家信息等关键参数。这些信息用于生成符合实际通信标准的设备标识码（如IMSI）。

## 配置文件结构

运营商信息存储在 `src/config/carrier_info.py` 文件中，使用字典结构组织数据：

```python
CARRIER_INFO = {
    "China": {
        "MCC": "460",
        "MNC": {
            "China Mobile": ["00", "02", "07"],
            "China Unicom": ["01", "06", "09"],
            "China Telecom": ["03", "05", "11"]
        },
        "PhonePrefix": ["139", "138", "137", ...]
    },
    ...
}
```

## 字段说明

### MCC (Mobile Country Code)
- **定义**：移动国家代码，由ITU-T分配给每个国家或地区
- **格式**：3位数字字符串
- **示例**：中国为"460"，美国为"310"

### MNC (Mobile Network Code)
- **定义**：移动网络代码，标识特定运营商
- **格式**：2-3位数字字符串，根据国家不同长度可能不同
- **示例**：中国移动在中国的MNC为"00","02","07"

### long_name
- **定义**：运营商的完整名称
- **格式**：字符串
- **示例**："China Mobile"，"Verizon"

### short_name
- **定义**：运营商的简称
- **格式**：字符串
- **示例**："CMCC"，"AT&T"

### 国家
- **定义**：运营商所在国家或地区的中文名称
- **格式**：字符串
- **示例**："中国"，"美国"

### 国家（英文）
- **定义**：国家或地区的ISO 3166-1字母代码
- **格式**：大写英文字母
- **示例**："CN"，"US"

### 国家编号
- **定义**：国家或地区的国际电话区号
- **格式**：数字字符串
- **示例**：中国为"+86"，美国为"+1"

### 号码长度
- **定义**：手机号码的总长度（不含国家代码）
- **格式**：整数
- **示例**：中国为11位，美国为10位

### 号码前缀
- **定义**：手机号码的前几位数字，用于区分不同的运营商或服务类型
- **格式**：字符串列表
- **示例**：中国为["139", "138", "137", ...]

## 配置规则

1. **MCC必须唯一**：每个国家/地区只能有一个MCC
2. **MNC不能重复**：同一国家内的不同运营商应有唯一的MNC
3. **国家编码标准化**：使用ISO 3166-1标准的国家代码
4. **号码前缀规范**：确保前缀与实际运营商使用的号码范围一致

## 维护流程

1. 确认新增运营商的MCC和MNC信息
2. 添加到CARRIER_INFO字典中
3. 更新相关函数以支持新运营商
4. 测试验证配置是否正确

## 常见问题

Q: 如何查找某个国家的MCC？
A: 可以参考ITU-T发布的官方MCC列表，或查询权威电信数据库。

Q: 同一运营商在不同国家的MNC是否相同？
A: 不同，MNC是国家特定的，同一运营商在不同国家会有不同的MNC。

Q: 如何添加新的运营商？
A: 在CARRIER_INFO字典中添加新的国家条目，然后在该国家的MNC字典中添加运营商及其对应的MNC列表。
